üîç What is Strict Mode?

Introduced in ECMAScript 5 (ES5), strict mode is a way to opt into a restricted variant of JavaScript, thereby implicitly opting out of "sloppy mode". It helps in catching common coding bloopers, prevents the use of potentially problematic features, and makes it easier to write "secure" JavaScript.

üöÄ Enabling Strict Mode

Strict mode can be enabled by adding the directive "use strict"; at the beginning of a script or a function. 

// For entire script
"use strict";
...

// For a specific function
function myFunction() {
  "use strict";
  ...
}

üß† Key Changes in Strict Mode

  • Eliminates some JavaScript silent errors by changing them to throw errors.
  • Fixes mistakes that make it difficult for JavaScript engines to perform optimizations.
  • Prohibits some syntax likely to be defined in future versions of ECMAScript.
  • Disallows the use of undeclared variables.
  • Prevents the use of duplicate parameter names.
  • Disallows assignments to read-only properties.
  • Prohibits the use of the with statement.
  • Changes the semantics of eval() and arguments.
  • In functions, this is undefined if the function is called without an object.

üìå Important Notes for Senior/Lead Developers

  • Modules and classes are in strict mode by default; no need to explicitly enable it.
  • Be cautious when integrating third-party libraries that may not be strict mode compliant.
  • Ensure that team members are aware of strict mode implications to maintain code consistency.
  • Use strict mode to catch common errors early in the development process, improving code quality.
  • Understand that strict mode can affect the behavior of this and eval(), which may impact legacy code.
