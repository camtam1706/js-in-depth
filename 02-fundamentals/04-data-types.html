üî¢ Primitive Types

  • Number: Represents both integer and floating-point numbers. Special values include NaN, Infinity, and -Infinity.
  • BigInt: Used for integers beyond the safe range of Number. Created by appending n to the end of an integer, e.g., 123n.
  • String: Represents textual data. Can use single, double, or backticks for template literals.
  • Boolean: Logical type with two values: true and false.
  • Null: Represents an intentional absence of any object value. Note: typeof null returns "object", which is a known quirk.
  • Undefined: Indicates a variable that has been declared but not assigned a value.
  • Symbol: Unique and immutable primitive value, often used as the key of an Object property to avoid property name collisions.

üß± Object Type

Objects are collections of key-value pairs. They can store multiple values and more complex entities. Functions, arrays, and other objects are all of the object type.

üîÑ Type Conversions

  • String Conversion: Occurs when outputting values. Can be performed with String(value).
  • Numeric Conversion: Occurs in mathematical operations. Can be performed with Number(value).
  • Boolean Conversion: Occurs in logical operations. Can be performed with Boolean(value).

üßë‚Äçüíº Notes for Senior/Lead Developers

  • Be cautious with type coercion, especially in comparisons. Use === and !== to avoid unexpected results.
  • Understand the differences between null and undefined, and use them appropriately to represent absence of value.
  • Leverage Symbol to create unique object property keys, especially when integrating with third-party code to avoid property name collisions.
  • Avoid using new Number(), new String(), or new Boolean() as they create wrapper objects, which can lead to unexpected behavior.
  • Be aware of the limitations of the Number type with large integers and consider using BigInt when precision is required.
