The alert() function displays a modal dialog with a message and an OK button. It pauses script execution until the user acknowledges the message.

The prompt() function displays a modal dialog with a message, an input field, and OK/Cancel buttons. It returns the input value or null if canceled.

‚úÖ confirm()

The confirm() function displays a modal dialog with a message and OK/Cancel buttons. It returns true if OK is pressed, and false otherwise.

üß† Summary

  • alert(): Displays a message. Pauses script execution until OK is pressed.
  • prompt(): Asks the user for input. Returns the input or null if canceled.
  • confirm(): Asks the user to confirm. Returns true for OK, false for Cancel.

Note: These dialogs are modal and block interaction with the rest of the page until dismissed. Their appearance and positioning are controlled by the browser and cannot be customized.

‚ö†Ô∏è Notes for Senior/Lead Developers

  • Use these dialogs sparingly in production environments, as they are blocking and can disrupt user experience.
  • Consider using custom modal implementations for better control over styling and behavior.
  • Be aware of browser inconsistencies, especially with prompt() in Internet Explorer, which may display "undefined" if the default value is not provided.
  • Always handle the possibility of null returns from prompt() and confirm() to avoid unexpected errors.
