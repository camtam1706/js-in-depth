🔄 JavaScript Type Conversions

🧠 Overview

JavaScript automatically converts values between types when necessary. Understanding these conversions helps prevent unexpected behaviors.

🔢 Numeric Conversion

  • Occurs in mathematical operations and comparisons.
  • Examples:
    • Number("123") → 123
    • Number("123z") → NaN
    • Number(true) → 1
    • Number(null) → 0
    • Number(undefined) → NaN

📝 String Conversion

  • Occurs when outputting values, such as with alert() or string concatenation.
  • Examples:
    • String(true) → "true"
    • String(null) → "null"
    • String(123) → "123"

✅ Boolean Conversion

  • Occurs in logical contexts, such as conditionals.
  • Falsy values:
    • 0, "", null, undefined, NaN
  • Truthy values:
    • All other values, including "0" and " "
  • Examples:
    • Boolean("0") → true
    • Boolean("") → false

🧑‍💼 Notes for Senior/Lead Developers

  • Be cautious with implicit type coercion; prefer explicit conversions for clarity.
  • Understand the behavior of == vs. === to prevent bugs.
  • Use utility functions to handle type conversions consistently across the codebase.
  • Educate team members about type conversion pitfalls to maintain code quality.
