  • Operands: The values that operators act upon. For example, in 5 * 2, 5 and 2 are operands.
  • Unary Operators: Operate on a single operand. Example: -x negates the value of x.
  • Binary Operators: Operate on two operands. Example: x + y adds x and y.

‚ûï Arithmetic Operators

  • +: Addition
  • -: Subtraction
  • *: Multiplication
  • /: Division
  • %: Remainder
  • **: Exponentiation

üßµ String Concatenation

The + operator can also concatenate strings. If either operand is a string, the other is converted to a string:

"Hello" + " World"      // "Hello World"
"1" + 2 + 2                  // "122"     (left-to-right: "1" + 2 ‚Üí "12", then "12" + 2 ‚Üí "122")
2 + 2 + "1"                  // "41"      (left-to-right: 2 + 2 ‚Üí 4, then 4 + "1" ‚Üí "41")
"Number: " + 3 + 4           // "Number: 34"
"10" - 2                    // 8         ("10" is coerced to number)
"10" * "2"                  // 20        (both strings are coerced to numbers)
"hello" * 2                // NaN       ("hello" cannot be converted to number)

üß† Note: Only the + operator triggers string concatenation. Other arithmetic operators (e.g., -, *) coerce operands to numbers.

üîç Comparison Operators

  • ==: Equality (with type coercion)
  • ===: Strict equality (no type coercion)
  • !=: Inequality (with type coercion)
  • !==: Strict inequality (no type coercion)
  • >, <, >=, <=: Greater than, less than, etc.

üîç Comparison & Logical Operators

  • Comparison: ==, !=, ===, !==, >, <, >=, <=
  • Logical: &&, ||, !
  • Nullish Coalescing: ?? returns the right-hand operand when the left is null or undefined.
  • Falsy values: These values evaluate to false in logical expressions: 
    false, 0, -0, "", null, undefined, NaN

ü§∑ Nullish Coalescing Operator

The ?? operator returns the first defined value (not null or undefined): 

let user;
alert(user ?? "Anonymous"); // "Anonymous"

üìù Assignment Operators

  • =: Assigns a value to a variable.
  • +=, -=, *=, /=, %=: Performs operation and assigns the result.

‚ùì Conditional (Ternary) Operator

Syntax: condition ? valueIfTrue : valueIfFalse

let accessAllowed = (age > 18) ? true : false;

üß† Senior/Lead Considerations

  • Type Coercion Awareness: Understand how JavaScript coerces types, especially with == vs. ===.
  • Short-Circuit Evaluation: Leverage || and && for default values and conditional execution.
  • Nullish Coalescing: Use ?? to provide default values only when dealing with null or undefined.
  • Operator Precedence: Be cautious with complex expressions; use parentheses to ensure correct evaluation order.
  • Code Readability: Avoid overusing the ternary operator in complex conditions; prefer if...else for clarity.
