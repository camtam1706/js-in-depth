üìò Basic Comparison Operators

  • > : Greater than
  • < : Less than
  • >= : Greater than or equal to
  • <= : Less than or equal to
  • == : Equal (loose equality, allows type coercion)
  • === : Strictly equal (no type coercion)
  • != : Not equal (loose inequality)
  • !== : Strictly not equal

üî§ String Comparison

Strings are compared lexicographically, based on Unicode values:


console.log('Z' > 'A'); // true
console.log('Glow' > 'Glee'); // true
console.log('Bee' > 'Be'); // true

üîÑ Comparison with Different Types

When comparing values of different types, JavaScript converts them to numbers:


console.log('2' > 1); // true
console.log('01' == 1); // true
console.log(true == 1); // true
console.log(false == 0); // true

‚ùì Comparison with null and undefined

Examples of how null and undefined behave in comparisons:


// Equality checks
console.log(null == undefined);  // true
console.log(null === undefined); // false

// Comparisons with numbers
console.log(null > 0);  // false
console.log(null == 0); // false
console.log(null >= 0); // true

‚ö†Ô∏è Be cautious: comparisons involving null or undefined can lead to unexpected results due to type coercion.

üß± Object Comparison

Objects are compared by reference, not by value:


let a = {};
let b = {};

console.log(a == b); // false

To compare object values, compare each property manually or use utility functions.

üß† Tips for Senior/Lead Developers

  • Prefer === and !== over == and != to avoid type coercion issues.
  • Be cautious when comparing null and undefined; consider using == null to check for both.
  • Remember that object comparisons check for reference equality, not value equality.
  • Use Number.isNaN() to accurately check for NaN values.
  • Understand JavaScript's type coercion rules to prevent unexpected comparison results.
