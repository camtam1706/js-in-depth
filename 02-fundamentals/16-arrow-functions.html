⚡ JavaScript Arrow Functions - Summary

Summarized using the 80/20 principle + key notes for Senior/Lead developers

📌 Key Takeaways (80/20 Principle)

1. Concise Syntax

Arrow functions are a shorter syntax for writing function expressions.

let sum = (a, b) => a + b;

Equivalent to:

let sum = function(a, b) {
  return a + b;
};

Single parameter:

let double = n => n * 2;

No parameters:

let sayHi = () => alert("Hello!");

2. Multi-line Functions

let sum = (a, b) => {
  let result = a + b;
  return result;
};

3. Advantages

  • Concise, ideal for callbacks and small functions.
  • Lexical this: inherits from outer scope.

4. Limitations

  • Cannot be used as constructors with new.
  • No own arguments object.
  • Not suitable for object methods due to lacking own this.

🚫 Arrow Functions and arguments

Arrow functions do not have their own arguments object. Unlike regular functions, you cannot access arguments directly within an arrow function.

📛 Problem Example:

const arrow = () => {
        console.log(arguments); // ❌ ReferenceError: arguments is not defined
      };
      
arrow(1, 2, 3);

✅ Solution: Use Rest Parameters

Instead of using arguments, use the modern ...args syntax:

const arrow = (...args) => {
        console.log(args); // ✅ Output: [1, 2, 3]
      };
      
arrow(1, 2, 3);

🧠 Senior/Lead Considerations

  • Don't assume arguments exists in arrow functions — always use ...args if needed.
  • Be careful when refactoring regular functions to arrow functions, especially in utility libraries or shared code.
  • In performance-critical code, rest parameters may have a slight overhead compared to native arguments — measure if necessary.

👨‍💼 Notes for Senior/Lead Developers

1. Controlling this

Arrow functions inherit this and should not be used for methods inside objects or classes where a specific this context is needed.

2. Readability & Maintainability

Do not overuse arrow functions for complex logic — can harm code clarity and team understanding.

3. Coding Standards Alignment

Ensure arrow function usage aligns with your team/project coding guidelines (e.g., use for simple callbacks only).

4. Performance & Compatibility

Make sure your runtime environment (e.g. Node.js version or older browsers) fully supports ES6 arrow functions before broad adoption.
