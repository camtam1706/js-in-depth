üß± Code Structure

  • Statements are usually separated by semicolons. JavaScript can insert them automatically (ASI), but this can lead to bugs.
  • Avoid relying on ASI‚Äîalways use semicolons to terminate statements explicitly.
  • Problematic cases include return statements, or lines beginning with [ or (.

‚úÖ Example ‚Äì Works correctly with semicolon:

function getValue() {
  return {
    name: "Cam"
  };
}
    
console.log(getValue()); // { name: "Cam" }

‚ùå Example ‚Äì Fails without semicolon (ASI inserts after return):

function getValue() {
  return
    {
      name: "Cam"
    };
}
    
console.log(getValue()); // undefined

The object is never returned because ASI inserts a semicolon right after return.

‚ùå Example ‚Äì Unexpected behavior when line starts with (:

let result = getValue()
    (function() {
      console.log("This is a function expression.");
    })();

Without a semicolon after getValue(), JavaScript interprets the following line as a function call on the result of getValue(), leading to a TypeError if getValue() doesn't return a function.

‚úÖ Corrected Example ‚Äì Adding semicolon to prevent misinterpretation:

let result = getValue();
    (function() {
      console.log("This is a function expression.");
    })();

By adding a semicolon after getValue(), the following function expression is correctly interpreted as a separate statement.

üîí Strict Mode

  • Enable strict mode by adding "use strict"; at the beginning of your scripts or functions.
  • Strict mode helps catch common coding errors and prevents the use of certain problematic features.
  • Some modern features, like classes, automatically enforce strict mode.

üì¶ Variables

  • Use let and const for variable declarations; avoid var due to its function-scoped behavior.
  • const is used for variables that should not be reassigned.
  • Variable names can include letters, digits (not as the first character), $, and _.
  • JavaScript is dynamically typed; variables can hold any type of value.

üî¢ Data Types

  • JavaScript has 8 basic data types: number, bigint, string, boolean, null, undefined, object, and symbol.
  • Use typeof to check the type of a variable, e.g., typeof 42 returns "number".
  • Note: typeof null returns "object", which is a known quirk in JavaScript.

üí¨ Interaction

  • alert(message): Displays a message to the user.
  • prompt(question, default): Asks the user for input and returns the entered value or null if canceled.
  • confirm(question): Asks the user to confirm something; returns true for OK and false for Cancel.

‚öôÔ∏è Operators

  • JavaScript includes standard arithmetic operators: +, -, *, /, %.
  • Comparison operators: ==, ===, !=, !==, >, <, >=, <=.
  • Logical operators: && (AND), || (OR), ! (NOT).
  • Use === and !== to avoid type coercion issues associated with == and !=.

üß† Senior/Lead Developer Considerations

  • Always use strict mode to enforce better coding practices and catch errors early.
  • Prefer const and let over var to avoid issues with variable hoisting and scope.
  • Be cautious with automatic semicolon insertion; always use semicolons to prevent unexpected behavior.
  • Understand the nuances of type coercion and use strict equality checks to avoid bugs.
  • Regularly review and adhere to coding standards and best practices to maintain code quality and readability.
