The Sources panel in Chrome DevTools is essential for debugging JavaScript. It consists of three main areas:

  • File Navigator: Lists all files loaded in the page, including HTML, CSS, and JavaScript.
  • Code Editor: Displays the source code of the selected file.
  • JavaScript Debugging Pane: Provides tools for setting breakpoints and stepping through code.

To access it, open DevTools with F12 (or Cmd+Opt+I on Mac) and select the Sources tab.

üß™ Using the Console

Press Esc to open the console at the bottom of DevTools. Here, you can execute JavaScript commands and inspect values.

For example: 

1 + 2 // Returns 3
hello("debugger") // Returns undefined

‚è∏Ô∏è Breakpoints

Breakpoints allow you to pause code execution at specific lines:

  • Click on the line number in the Code Editor to set a breakpoint.
  • Use the right panel to manage breakpoints: enable/disable or remove them.
  • Right-click a line number to set a conditional breakpoint that triggers only when a specified condition is true.

üß≠ The ‚Äúdebugger‚Äù Statement

Insert the debugger; statement in your code to pause execution when DevTools is open: 

function hello(name) {
  let phrase = `Hello, ${name}!`;

  debugger; // Execution pauses here

  say(phrase);
}

This is useful for debugging without manually setting breakpoints.
