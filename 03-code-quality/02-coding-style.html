üìê Syntax Rules

  • Use "Egyptian" style braces: opening brace on the same line as the statement.
  • Prefer braces even for single-line blocks to enhance readability.
  • Limit line length to 80‚Äì120 characters for better readability.
  • Use 2 spaces for indentation; spaces are preferred over tabs.
  • Insert empty lines to separate logical blocks of code.
  • Always use semicolons to terminate statements.

Example:

function pow(x, n) {
  let result = 1;

  for (let i = 0; i < n; i++) {
    result *= x;
  }

  return result;
}

üß∞ Function Placement

There are three common ways to organize functions:

  1. Functions before usage: Define all functions at the top.
  2. Functions after usage: Place helper functions below the main code.
  3. Inline functions: Define functions where they are first used.

The second approach is often preferred as it allows readers to understand the main logic before diving into helper functions.

Example:

// Main code
let elem = createElement();
setHandler(elem);
walkAround();

// Helper functions
function createElement() {
  // ...
}

function setHandler(elem) {
  // ...
}

function walkAround() {
  // ...
}

üìö Style Guides

Adopting a consistent style guide ensures uniformity across the codebase. Popular style guides include:

  • Google JavaScript Style Guide
  • Airbnb JavaScript Style Guide
  • Idiomatic.JS
  • StandardJS

Choose one that fits your team's preferences and stick to it.

Example from Airbnb Style Guide:

// Use const for all of your references; avoid using var
const a = 1;
const b = 2;

// Use arrow functions
const add = (x, y) => x + y;

üßπ Automated Linters

Linters help maintain code quality by enforcing style rules and catching potential errors. Popular linters include:

  • JSLint: One of the first JavaScript linters.
  • JSHint: Offers more configuration options than JSLint.
  • ESLint: Highly configurable and widely adopted.

To set up ESLint:

  1. Install Node.js.
  2. Install ESLint globally: npm install -g eslint
  3. Create a configuration file .eslintrc in your project root.
  4. Configure your editor to integrate with ESLint.

Example .eslintrc configuration:

{
  "extends": "eslint:recommended",
  "env": {
    "browser": true,
    "node": true,
    "es6": true
  },
  "rules": {
    "no-console": 0,
    "indent": ["error", 2]
  }
}
