  • Brevity is the soul of wit: Using complex ternary operators to make code concise but hard to read.

  • One-letter variables: Employing single-letter variable names like a, b, or c to obscure meaning. 
    // Example:
function calc(a, b) {
  return a * b;
}
  • Use abbreviations: Shortening variable names (e.g., list to lst) to reduce clarity. 
    // Example:

  • Soar high. Be abstract.: Using vague names like data, value, or item that lack specific meaning. 
    // Example:
function process(data) {
  return data.map(item => doSomething(item));
}
  • Overlap outer variables: Re-declaring variables within inner scopes, leading to confusion and potential bugs. 
    // Example:
let value = 10;
function compute() {
  let value = 20;
  return value;
}
  • Side-effects everywhere!: Writing functions that appear pure but have hidden side effects, making debugging difficult. 
    // Example:
let total = 0;
function addToTotal(a) {
  total += a;
  return total;
}
