Optional Chaining (?.) allows safe access to nested object properties without checking every level manually.

üõ†Ô∏è Usage

Optional chaining (`?.`) is a powerful feature in JavaScript that allows for safe access to nested object properties, methods, or array elements. It prevents runtime errors when accessing properties of `undefined` or `null` values.

‚úÖ When to Use

  • Accessing deeply nested properties where intermediate properties might be `null` or `undefined`. 
    let user = {};
console.log(user?.address?.street); // undefined, no error
  • Calling methods that may not exist. 
    user.sayHi?.(); // No error if sayHi doesn't exist
  • Accessing array elements when the array might be `null` or `undefined`. 
    let users = null;
console.log(users?.[0]); // undefined, no error

‚ùå When Not to Use

  • Accessing properties of objects that are guaranteed to exist. 
    let user = { name: "Alice" };
console.log(user.name); // "Alice"
console.log(user?.name); // "Alice" - unnecessary use of ?.
  • When the variable itself is not declared. Using `?.` does not prevent a `ReferenceError`. 
    console.log(user?.name); // ReferenceError: user is not defined
  • On the left-hand side of an assignment. Optional chaining cannot be used to assign values. 
    let user = null;
user?.name = "John"; // SyntaxError

Use optional chaining judiciously to maintain code clarity and avoid masking potential bugs.

‚ö†Ô∏è Cautions

  • Only use when it's okay that the object might be null or undefined.
  • Overuse can hide bugs.
  • The variable before ?. must be declared:
console.log(user?.name); // ReferenceError: user is not defined

To avoid this, always ensure the variable exists before using ?..

‚úçÔ∏è Optional Chaining and Assignments

The optional chaining operator cannot be used on the left-hand side of an assignment because it's not an L-value. 

let user = null;

// ‚ùå Invalid
user?.name = "John"; // SyntaxError

// ‚úÖ Valid workaround
if (user) {
  user.name = "John";
}

Optional chaining is only for reading, not writing values.
