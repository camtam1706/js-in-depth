📚 JavaScript Strings Summary

🔤 String Basics

  • Strings are sequences of characters enclosed in single quotes (' '), double quotes (" "), or backticks (` `).
  • Backticks allow for template literals, enabling expression interpolation and multi-line strings.
  • Example: 
    let name = 'John';
let greeting = `Hello, ${name}!`;

🔧 Special Characters

  • Use escape sequences to represent special characters:
    • \n – New line
    • \t – Tab
    • \\ – Backslash
    • \', \", \` – Quotes
  • Example: 
    let message = "Line1\nLine2";

🛠️ String Properties and Methods

  • length – Returns the number of characters in a string.
  • toUpperCase(), toLowerCase() – Change the case of the string.
  • indexOf(substring) – Returns the index of the first occurrence of the substring.
  • includes(substring) – Checks if the substring exists within the string.
  • slice(start, end) – Extracts a section of the string.
  • substring(start, end) – Similar to slice but doesn't accept negative indices.
  • substr(start, length) – Extracts a substring based on length.

⚖️ String Comparison

  • Strings are compared based on lexicographical order using Unicode values.
  • Example: 
    'apple' > 'banana' // false

🌐 Advanced Unicode Handling

  • JavaScript strings are UTF-16 encoded, meaning each character is represented by one or two 16-bit code units.
  • Characters outside the Basic Multilingual Plane (BMP) are represented using surrogate pairs.
  • Use codePointAt() to accurately retrieve the Unicode code point of characters, especially those represented by surrogate pairs.
  • Example: 
    const emoji = '😊';
console.log(emoji.codePointAt(0)); // 128522
  • String comparison is based on Unicode code points. For instance: 
    console.log('A' < 'a'); // true
console.log('a' < 'ä'); // true
console.log('ä' < 'z'); // false
    This is because:
    • 'A': U+0041 (65)
    • 'a': U+0061 (97)
    • 'ä': U+00E4 (228)
    • 'z': U+007A (122)
  • For locale-aware comparisons, use localeCompare(): 
    console.log('ä'.localeCompare('z', 'de')); // -1 (in German locale)

🧠 Senior/Lead Developer Considerations

  • Be cautious with string immutability; operations return new strings.
  • Understand the implications of UTF-16 encoding, especially with emojis and special characters.
  • Avoid using eval() for executing code within strings due to security risks.
  • Utilize template literals for cleaner and more readable code.
  • Be aware of differences between slice(), substring(), and substr().
