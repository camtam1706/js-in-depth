📚 JavaScript Map & Set

🗺️ Map Overview

  • Map is a collection of keyed data items, similar to an Object, but allows keys of any type.
  • Methods:
    • new Map() – creates the map.
    • map.set(key, value) – stores the value by the key.
    • map.get(key) – returns the value by the key.
    • map.has(key) – returns true if the key exists.
    • map.delete(key) – removes the element by the key.
    • map.clear() – removes everything from the map.
    • map.size – returns the current element count.
  • Keys are not converted to strings; any type of key is possible.
  • Chaining is possible: map.set('1', 'str1').set(1, 'num1').set(true, 'bool1');

🔢 Set Overview

  • Set is a collection of unique values.
  • Methods:
    • new Set() – creates the set.
    • set.add(value) – adds a value.
    • set.has(value) – checks for the value.
    • set.delete(value) – removes the value.
    • set.clear() – clears the set.
    • set.size – returns the number of elements.
  • Useful for storing unique values of any type.
  • Can be converted to an array: Array.from(set)

⚖️ Map vs Object

  • Map allows keys of any type, maintains insertion order, and has a size property.
  • Object keys are strings or symbols, and does not guarantee order.
  • Use Map when you need keys of various types and order matters.
  • Use Object when you need simple key-value pairs with string keys.

🔁 Set vs Array

  • Set stores unique values, automatically removing duplicates.
  • Array allows duplicate values and provides more methods for manipulation.
  • Use Set when uniqueness is required.
  • Convert between them as needed: Array.from(set) and new Set(array)

🔄 For Loop vs Map & Set Iteration

  • For Loop:
    • Higher performance in most cases, especially with large datasets.
    • Allows tight control over the iteration flow, including the use of break and continue.
    • Suitable when complex operations are needed in each iteration.
  • Map & Set Iteration:
    • Uses for...of or forEach to iterate through elements.
    • More readable and concise, especially when handling simple operations.
    • Preserves the original insertion order of elements.
  • Performance Comparison:
    • The for loop is typically faster than forEach or map due to no overhead from callback functions.
    • With small datasets, the performance difference is negligible; prioritize writing code that is easy to read and maintain.

🧠 Senior/Lead Considerations

  • Understand the differences between Map, Set, WeakMap, and WeakSet.
  • Use WeakMap and WeakSet for memory-sensitive applications, as they allow garbage collection of keys.
  • Be aware of the performance implications of using these data structures in large-scale applications.
  • Ensure proper iteration and conversion when interfacing with APIs that expect arrays or objects.
