üì¶ JavaScript Destructuring Assignment

üìò Overview

Destructuring assignment is a syntax in JavaScript that allows unpacking values from arrays or properties from objects into distinct variables. It provides a concise way to extract multiple properties or elements in a single statement.

üî¢ Array Destructuring

  • Basic syntax: const [a, b] = [1, 2];
  • Skipping elements: const [a, , b] = [1, 2, 3];
  • Default values: const [a = 1, b = 2] = [];
  • Rest operator: const [a, ...rest] = [1, 2, 3];
  • Swapping variables: [a, b] = [b, a];
  • Works with iterables: const [a, b] = 'hi';
// Basic array destructuring
const [firstName, surname] = ["John", "Smith"];
console.log(firstName); // "John"
console.log(surname);   // "Smith"

// Skipping elements
const [firstName, , title] = ["Julius", "Caesar", "Consul"];
console.log(title); // "Consul"

// Using rest operator
const [name1, name2, ...rest] = ["Julius", "Caesar", "Consul", "of the Roman Republic"];
console.log(rest); // ["Consul", "of the Roman Republic"]

// Swapping variables
let a = 1, b = 2;
[a, b] = [b, a];
console.log(a); // 2
console.log(b); // 1

// Destructuring strings
const [a, b, c] = "abc";
console.log(a); // "a"
console.log(b); // "b"
console.log(c); // "c"

üß© Object Destructuring

  • Basic syntax: const {a, b} = obj;
  • Renaming variables: const {a: newA} = obj;
  • Default values: const {a = 1} = {};
  • Nested destructuring: const {a: {b}} = obj;
  • Rest operator: const {a, ...rest} = obj;
  • Dynamic property names: const {[key]: value} = obj;
// Basic object destructuring
const user = { name: "John", age: 30 };
const { name, age } = user;
console.log(name); // "John"
console.log(age);  // 30

// Renaming variables
const { name: userName } = user;
console.log(userName); // "John"

// Default values
const { isAdmin = false } = user;
console.log(isAdmin); // false

// Nested destructuring
const nestedUser = { name: "John", address: { city: "New York" } };
const { address: { city } } = nestedUser;
console.log(city); // "New York"

// Rest operator
const { name: n, ...rest } = user;
console.log(n);    // "John"
console.log(rest); // { age: 30 }

üõ†Ô∏è Function Parameters

Destructuring can be used directly in function parameters to extract specific values: 

function greet({ name, age }) {
  console.log(`Hello, ${name}. You are ${age} years old.`);
}

greet({ name: "Alice", age: 25 }); // "Hello, Alice. You are 25 years old."

This approach is especially useful for functions that receive configuration objects.

üë®‚Äçüíº Senior/Lead Considerations

  • Readability: Avoid overusing nested destructuring which can reduce code clarity.
  • Default values: Ensure default values are set appropriately to prevent undefined errors.
  • Function parameters: Be cautious when destructuring in function parameters, especially when the function might be called with undefined or null arguments.
  • Consistency: Maintain consistent use of destructuring across the codebase for better maintainability.
  • Performance: While destructuring is convenient, be aware of its performance implications in critical code paths.
