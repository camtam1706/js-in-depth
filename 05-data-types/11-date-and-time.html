üõ†Ô∏è Creating Dates

JavaScript provides multiple ways to create Date objects:

  • new Date(): Current date and time. 
    const now = new Date();
console.log(now);
  • new Date(milliseconds): Milliseconds since Jan 1, 1970 UTC. 
    const date = new Date(86400000); // Jan 2, 1970
console.log(date);
  • new Date(dateString): Parses ISO 8601 strings. 
    const date = new Date("2023-12-25T10:00:00");
console.log(date);
  • new Date(year, monthIndex, day, hours, minutes, seconds, milliseconds): Months are 0-based. 
    const date = new Date(2023, 11, 25, 10, 0, 0);
console.log(date);

üìÜ Accessing Components

Retrieve specific components from a Date object:

  • getFullYear(): Four-digit year. 
    const date = new Date("2021-03-25");
console.log(date.getFullYear()); // 2021
  • getMonth(): Month (0-11). 
    console.log(date.getMonth()); // 2 (March)
  • getDate(): Day of the month (1-31). 
    console.log(date.getDate()); // 25
  • getDay(): Day of the week (0-6). 
    console.log(date.getDay()); // 4 (Thursday)
  • getHours(), getMinutes(), getSeconds(), getMilliseconds(): Time components. 
    console.log(date.getHours()); // 0
console.log(date.getMinutes()); // 0
console.log(date.getSeconds()); // 0
console.log(date.getMilliseconds()); // 0
  • getUTC*: UTC equivalents of the above methods. 
    console.log(date.getUTCFullYear()); // 2021
  • getTimezoneOffset(): Difference between UTC and local time in minutes. 
    console.log(date.getTimezoneOffset()); // e.g., -420

üß≠ Modifying Dates

Modify components of a Date object:

  • setFullYear(), setMonth(), setDate(), etc. 
    const date = new Date();
date.setFullYear(2025);
date.setMonth(0); // January
date.setDate(1);
console.log(date);
  • Auto-corrects invalid dates: 
    const date = new Date(2023, 1, 30); // Feb 30
console.log(date); // Mar 2, 2023
  • Negative or zero values adjust date accordingly: 
    const date = new Date(2023, 0, 0); // Dec 31, 2022
console.log(date);

‚è±Ô∏è Timestamps & Differences

Work with timestamps and calculate differences:

  • getTime() or unary plus: Milliseconds since Jan 1, 1970 UTC. 
    const date = new Date();
console.log(date.getTime());
console.log(+date);
  • Date.now(): Current timestamp without creating a Date object. 
    console.log(Date.now());
  • Subtracting dates yields difference in milliseconds: 
    const start = new Date("2023-01-01");
const end = new Date("2023-12-31");
const diff = end - start;
console.log(diff); // milliseconds
console.log(diff / (1000 * 60 * 60 * 24)); // days

üìà Benchmarking

Measure performance using timestamps:

  • Use Date.now() for performance measurements: 
    const start = Date.now();
// Code to measure
const end = Date.now();
console.log(`Elapsed time: ${end - start} ms`);
  • Run benchmarks multiple times to get reliable results.
  • Be cautious: JavaScript engines optimize hot code paths.

üß† For Senior/Lead Developers

Advanced considerations for date handling:

  • Always use getFullYear() instead of deprecated getYear().
  • Be aware of time zone differences and daylight saving time when manipulating dates.
  • Use performance.now() for high-resolution time measurements in browsers: 
    const start = performance.now();
// Code to measure
const end = performance.now();
console.log(`Elapsed time: ${end - start} ms`);
  • Consider using libraries like Moment.js or date-fns for complex date manipulations.
  • Ensure consistent date handling across different locales and time zones in applications.
