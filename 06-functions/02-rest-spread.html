ğŸ”§ Rest Parameters

Rest parameters allow functions to accept an indefinite number of arguments as an array. This is useful when the number of arguments is unknown. 

function sumAll(...args) {
  return args.reduce((sum, current) => sum + current, 0);
}

sumAll(1, 2, 3); // 6

ğŸ“¤ Spread Syntax

Spread syntax allows an iterable such as an array to be expanded in places where zero or more arguments or elements are expected. 

const numbers = [1, 2, 3];
const newNumbers = [...numbers, 4, 5]; // [1, 2, 3, 4, 5]

function multiply(a, b, c) {
  return a * b * c;
}

multiply(...numbers); // 6

  • Rest Parameters: Collects all remaining elements into an array.
  • Spread Syntax: Spreads an array into individual elements.
  • Both use the ... syntax but serve opposite purposes.

ğŸ§  Notes for Senior/Lead Developers

  • Ensure clarity between rest parameters and the arguments object; prefer rest parameters for cleaner syntax and better readability.
  • When using spread syntax, be cautious with deep cloning; spread creates shallow copies.
  • In performance-critical applications, assess the impact of using rest and spread, as they can introduce overhead.
  • Educate team members on the differences and appropriate use cases to maintain code consistency.
