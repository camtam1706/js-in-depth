üìò What is a Closure?

A closure is a function that retains access to its lexical scope, even when the function is executed outside that scope. This means the function "remembers" the environment in which it was created.

üß† Key Concepts

  • Lexical Scoping: Functions access variables from the scope in which they were defined, not where they are executed.
  • Function Factories: Functions can return other functions, each with their own preserved scope.
  • Data Encapsulation: Closures allow for private variables and methods, emulating object-oriented principles.

üí° Practical Examples

// Example: Counter using Closure
function createCounter() {
  let count = 0;
  return function() {
    count++;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2

üöÄ Benefits of Using Closures

  • State Preservation: Maintain state between function calls without using global variables.
  • Encapsulation: Create private variables and functions.
  • Higher-Order Functions: Enable functions that return other functions or take them as arguments.
