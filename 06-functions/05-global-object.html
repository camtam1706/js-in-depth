🌐 JavaScript Global Object Overview

📌 What is the Global Object?

The global object provides variables and functions that are available anywhere. By default, those that are built into the language or the environment. In a browser it is named window, for Node.js it is global, for other environments it may have another name. Recently, globalThis was added to the language, as a standardized name for a global object, that should be supported across all environments. It’s supported in all major browsers.

🧠 Key Characteristics

  • Variables declared with var in the global scope become properties of the global object.
  • Variables declared with let and const do not become properties of the global object.
  • Functions declared in the global scope become methods of the global object.
  • Using global variables is generally discouraged. There should be as few global variables as possible.

🔧 Practical Usage

You can access global functions and variables directly: 

var gVar = 5;
alert(window.gVar); // 5

Or define global properties explicitly: 

window.currentUser = { name: "John" };
alert(window.currentUser.name); // John

🚫 Best Practices

  • Minimize the use of global variables to reduce potential conflicts and maintain cleaner code.
  • Prefer using modules and proper scoping to encapsulate variables and functions.
  • Use globalThis for a consistent reference to the global object across different environments.

👨‍💼 Notes for Senior/Lead Developers

  • Ensure that your team understands the implications of polluting the global namespace and enforces best practices to avoid it.
  • Implement code reviews that check for unnecessary global variable declarations.
  • Encourage the use of modern JavaScript features like modules to promote better code organization and maintainability.
