The new Function() constructor allows creating functions dynamically from strings at runtime. This is particularly useful when the function body is not known until execution, such as receiving code from a server. 

let sum = new Function('a', 'b', 'return a + b');
alert( sum(1, 2) ); // 3

‚ö†Ô∏è Scope & Closure

Functions created with new Function do not capture the current lexical environment. Instead, their scope is the global environment. 

function getFunc() {
  let value = "test";
  let func = new Function('alert(value)');
  return func;
}
getFunc()(); // ReferenceError: value is not defined

In contrast, regular functions capture their lexical environment: 

function getFunc() {
  let value = "test";
  let func = function() { alert(value); };
  return func;
}
getFunc()(); // "test"

üîç Use Cases

  • Dynamic code execution received from external sources.
  • Compiling functions from templates in complex web applications.
  • Creating functions where the body is not known until runtime.

üö´ Limitations

  • Cannot access variables from the outer (local) scope.
  • Potential security risks if executing untrusted code.
  • May interfere with code minification and optimization processes.

üßë‚Äçüíº Notes for Senior/Lead Developers

  • Evaluate the necessity of using new Function; prefer safer alternatives when possible.
  • Ensure that any dynamically created code is sanitized to prevent security vulnerabilities.
  • Be aware of the implications on scope and closures, especially in larger codebases.
  • Consider the impact on performance and maintainability when introducing dynamic function creation.
