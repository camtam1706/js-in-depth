üìò Overview

JavaScript properties come in two forms: data properties and accessor properties. Accessor properties use getter and setter functions to control access to an object's properties, allowing for encapsulation and computed values.

‚öôÔ∏è Getters and Setters

Getters and setters are defined using the get and set keywords within object literals or classes. They allow you to execute code when a property is accessed or assigned. 

const user = {
  firstName: "John",
  lastName: "Doe",
  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  },
  set fullName(name) {
    [this.firstName, this.lastName] = name.split(" ");
  }
};

console.log(user.fullName); // "John Doe"
user.fullName = "Jane Smith";
console.log(user.firstName); // "Jane"
console.log(user.lastName); // "Smith"

üõ†Ô∏è Accessor Descriptors

Accessor properties can also be defined using Object.defineProperty, specifying get and set functions along with enumerable and configurable attributes. 

const user = {
  firstName: "John",
  lastName: "Doe"
};

Object.defineProperty(user, "fullName", {
  get() {
    return `${this.firstName} ${this.lastName}`;
  },
  set(name) {
    [this.firstName, this.lastName] = name.split(" ");
  },
  enumerable: true,
  configurable: true
});

console.log(user.fullName); // "John Doe"

üöÄ Best Practices

  • Use getters and setters to encapsulate internal property logic.
  • Avoid heavy computations or side effects within accessors.
  • Use clear and consistent naming conventions.
  • Document accessor behavior to aid maintainability.

üë®‚Äçüíº Considerations for Senior/Lead Developers

  • Ensure accessors do not introduce unexpected side effects that can complicate debugging.
  • Be cautious with performance implications of accessors in performance-critical code paths.
  • Establish and enforce coding standards regarding the use of accessors within your team.
  • Consider the implications of accessors on serialization and cloning of objects.
