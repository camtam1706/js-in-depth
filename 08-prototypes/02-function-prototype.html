üîß What is F.prototype?

In JavaScript, every function has a prototype property. When a function is used as a constructor (with the new keyword), the prototype property of that function is assigned as the [[Prototype]] of the newly created object.

Example: 

function Rabbit(name) {
  this.name = name;
}
Rabbit.prototype = { eats: true };

let rabbit = new Rabbit("White Rabbit");
console.log(rabbit.eats); // true

Here, rabbit inherits the eats property from Rabbit.prototype.

üìå Key Concepts

  • F.prototype is only used when new F() is called. It sets the [[Prototype]] of the new object.
  • Changing F.prototype after creating an object does not affect the existing object‚Äôs prototype.
  • Every function has a default prototype object with a constructor property pointing back to the function itself.
  • Overwriting the default prototype object removes the constructor property unless it is manually added back.

üß™ Practical Examples

Consider the following scenarios: 

function Rabbit() {}
Rabbit.prototype = { eats: true };

let rabbit = new Rabbit();
console.log(rabbit.eats); // true

Rabbit.prototype = {};
console.log(rabbit.eats); // true (existing object not affected)

Rabbit.prototype.eats = false;
console.log(rabbit.eats); // true (still refers to old prototype)

delete rabbit.eats;
console.log(rabbit.eats); // true (property exists on prototype)

delete Rabbit.prototype.eats;
console.log(rabbit.eats); // undefined (property deleted from prototype)

These examples illustrate how changes to the prototype affect objects created before and after the change.

üë®‚Äçüíº Notes for Senior/Lead Developers

  • Be cautious when modifying the prototype of constructors, especially in shared or third-party codebases, as it can lead to unexpected behaviors.
  • Always ensure that the constructor property is correctly set when overwriting the default prototype object to maintain proper references.
  • Understand the distinction between F.prototype (a property on the constructor function) and [[Prototype]] (an internal property of objects) to avoid confusion.
  • Use Object.create for creating objects with a specified prototype without invoking a constructor function, providing more control over the object's structure.
