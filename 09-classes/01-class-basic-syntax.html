üî§ Class Basic Syntax

Classes in JavaScript are syntactic sugar over the existing prototype-based inheritance. They provide a clearer and more concise syntax for creating objects and dealing with inheritance. 

class User {
  constructor(name) {
    this.name = name;
  }
  sayHi() {
    alert(this.name);
  }
}

let user = new User("John");
user.sayHi();

üß¨ Class Inheritance

JavaScript classes support inheritance through the extends keyword. A subclass can override methods from its superclass and use super to call the superclass's constructor or methods. 

class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    alert(`${this.name} makes a noise.`);
  }
}

class Dog extends Animal {
  speak() {
    super.speak();
    alert(`${this.name} barks.`);
  }
}

let dog = new Dog("Rex");
dog.speak();

‚öôÔ∏è Static Methods and Properties

Static members are defined on the class itself, not on instances. They are useful for utility functions. 

class MathHelper {
  static square(x) {
    return x * x;
  }
}

console.log(MathHelper.square(4)); // 16

üîí Private and Protected Members

JavaScript supports private fields using the # syntax. These fields are only accessible within the class. 

class Counter {
  #count = 0;
  increment() {
    this.#count++;
    console.log(this.#count);
  }
}

let counter = new Counter();
counter.increment(); // 1
// counter.#count; // SyntaxError

üß± Extending Built-in Classes

You can extend built-in classes like Array to create custom classes with additional functionality. 

class MyArray extends Array {
  sum() {
    return this.reduce((a, b) => a + b, 0);
  }
}

let arr = new MyArray(1, 2, 3);
console.log(arr.sum()); // 6

üîç Class Checking with instanceof

The instanceof operator checks if an object is an instance of a class or its subclasses. 

class Person {}
let person = new Person();

console.log(person instanceof Person); // true

üß© Mixins

Mixins allow objects to share functionality without using inheritance. 

let sayHiMixin = {
  sayHi() {
    alert("Hello");
  }
};

class User {}
Object.assign(User.prototype, sayHiMixin);

let user = new User();
user.sayHi(); // Hello

üß† Notes for Senior/Lead Developers

  • Design Patterns: Encourage the use of composition over inheritance where appropriate to promote code reuse and flexibility.
  • Encapsulation: Utilize private fields to encapsulate internal implementation details and expose only necessary interfaces.
  • Performance: Be cautious with extending built-in classes, as it may have performance implications in some JavaScript engines.
  • Consistency: Maintain consistent coding standards and practices across the codebase to improve readability and maintainability.
  • Tooling: Leverage TypeScript or JSDoc for better type safety and developer experience.
