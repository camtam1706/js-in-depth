‚ö° 1. What is a Callback?

A callback is a function passed into another function as an argument to be executed later. This allows asynchronous behavior, code reuse, and event handling in JavaScript.

Example: 

function greet(name) {
  alert(`Hello, ${name}!`);
}

function processUserInput(callback) {
  let name = prompt('Please enter your name.');
  callback(name);
}

processUserInput(greet);

üîÑ 2. Synchronous and Asynchronous Callbacks

Callbacks can be:

  • Synchronous: executed immediately during the function call.
  • Asynchronous: executed later, e.g., after a timeout, I/O event, or HTTP request.

Example of synchronous callback:

function sayHello(name) {
  alert(`Hello, ${name}!`);
}

function process(callback) {
  callback('Alice');
}

process(sayHello);

Example of asynchronous callback:

setTimeout(() => alert('Done!'), 1000);

üìã 3. Callback Hell and Pyramid of Doom

When callbacks are nested deeply, code becomes hard to read and maintain. This is called Callback Hell or the Pyramid of Doom. 

doSomething(function(result) {
  doSomethingElse(result, function(newResult) {
    doThirdThing(newResult, function(finalResult) {
      console.log('Got the final result: ' + finalResult);
    });
  });
});

To avoid this, modern JavaScript uses Promises and async/await syntax.

üéØ 4. Passing Arguments to Callbacks

You can pass any arguments you want to a callback function when calling it. 

function repeat(n, action) {
  for (let i = 0; i < n; i++) {
    action(i);
  }
}

repeat(3, console.log);
// Outputs: 0 1 2

Callbacks are essential for event-driven programming in JavaScript, especially in browser environments. You attach callbacks as event listeners that execute when an event occurs. 

button.addEventListener('click', function() {
  alert('Button clicked!');
});

‚ö†Ô∏è 6. Common Mistakes with Callbacks

  • Calling the callback immediately instead of passing a function reference.
  • Forgetting to handle errors properly in asynchronous callbacks.
  • Deeply nested callbacks leading to callback hell.

üöÄ Senior/Lead Must-Know Points

  • Maintainability: Avoid deeply nested callbacks by using Promises or async/await. Lead code reviews with focus on callback readability.
  • Error Handling: Ensure proper error propagation in callbacks; promote consistent patterns for error-first callbacks in Node.js style.
  • Performance: Understand the event loop and callback queue to avoid blocking UI or server event handling.
  • Testing: Ensure asynchronous callbacks are covered in unit and integration tests, avoiding flaky tests.
  • Code Standards: Advocate for use of modern async patterns (Promises, async/await) over legacy callbacks where possible.
  • Mentorship: Train juniors on callback fundamentals and pitfalls, emphasizing clear documentation and modular code.
