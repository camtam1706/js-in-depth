üó∫Ô∏è Overview

  • To find elements in the DOM, use getElementById, getElementsBy*, querySelector, and querySelectorAll.
  • Methods work on document or any DOM element (to search within a subtree).
  • All searching methods return ‚Äúlive‚Äù collections except querySelectorAll (which returns a static NodeList).

üÜî getElementById

  • Searches for an element by its id attribute. Returns the element or null if not found.
  • Syntax: let elem = document.getElementById('id');
  • Only available on the document object.

üè∑Ô∏è getElementsByTagName

  • Searches for elements by tag name (e.g., 'div', 'input').
  • Returns a live HTMLCollection of found elements.
  • Can be called on document or any DOM element.
  • Syntax: let elems = document.getElementsByTagName('tag')
  • Special case: * returns all elements in the subtree.

üè∑Ô∏è getElementsByClassName

  • Searches for elements by their class name.
  • Returns a live HTMLCollection.
  • Syntax: let elems = document.getElementsByClassName('class')
  • Can specify multiple classes separated by spaces.

üîó querySelector & querySelectorAll

  • querySelector: Returns the first element matching a CSS selector.
    • Syntax: let elem = document.querySelector('selector')
  • querySelectorAll: Returns all elements matching a CSS selector as a static NodeList.
    • Syntax: let elems = document.querySelectorAll('selector')
  • Selectors can be any valid CSS selector, including complex ones (e.g., div.menu > .item.selected).
  • You can loop through the NodeList with forEach or for..of.
  • These methods are available on both document and any DOM element.

‚ö° Live vs. Static Collections

  • Live collections (HTMLCollection from getElementsBy*) reflect the current DOM state and update automatically.
  • Static collections (NodeList from querySelectorAll) do not update when the DOM changes.
  • Choose static collections if you want a snapshot, live collections if you want auto-updates.

üîÅ Looping Through Results

  • HTMLCollection and NodeList can be iterated using for..of, but only NodeList (from querySelectorAll) supports forEach directly.
  • For HTMLCollection, convert to array: Array.from(collection).forEach(...)

üì¶ Searching within a Subtree

  • All methods except getElementById can be called on elements, not just document.
  • This allows searching within a specific section of the DOM tree (e.g., menu.querySelectorAll('.item')).

üö¶ Senior/Lead: Key Considerations

  • Performance: Prefer specific selectors and scoped queries to minimize the search space and improve performance, especially in large DOM trees.
  • Maintainability: Use querySelector/querySelectorAll for flexibility and clarity with complex selectors, but avoid overcomplicated selectors that are fragile to DOM changes.
  • Live Collections Caution: Avoid relying on live HTMLCollections for state-sensitive UI logic, as mutations can create subtle bugs. Favor static NodeLists when possible for predictable behavior.
  • Encapsulation: Always scope queries to components or containers to prevent side-effects and accidental selection of unintended nodes, especially in large apps or component-based architectures.
  • Testing: Consistently use data attributes (e.g., data-testid) for reliable element targeting in automated tests instead of relying solely on CSS classes or tag names.
  • Frameworks: Understand that most modern frameworks (React, Vue, Angular, etc.) abstract direct DOM searches, but knowing these methods is still critical for low-level manipulation, custom components, or performance tuning.
  • Browser Support: querySelector* is widely supported, but legacy codebases might use older methods; plan refactors carefully.
