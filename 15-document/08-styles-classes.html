üí° Introduction

JavaScript can dynamically change the appearance of elements on the page by manipulating their CSS styles and class attributes. Understanding how to read, set, and manage styles/classes from scripts is key for building interactive and dynamic UIs.

üñåÔ∏è The element.style Property

  • The style property allows you to read or set the **inline styles** of an element.
  • Example: elem.style.backgroundColor = "red";
  • Property names in JavaScript use camelCase, e.g., backgroundColor for background-color.
  • Only inline styles can be read/set via element.style. Styles from CSS classes or external stylesheets are not visible here.
// Example
<div id="myDiv" style="color: red">Hello</div>

let div = document.getElementById('myDiv');
alert(div.style.color); // "red"

üîç Getting Computed Styles

  • To get the **actual computed style** of an element (from all sources: inline, classes, external CSS), use getComputedStyle(element).
  • Returns a read-only object with all CSS properties in dash-case (e.g., background-color).
  • Example:
    let style = getComputedStyle(element);
    alert(style.marginTop);
  • For pseudo-elements, pass a second argument, e.g., getComputedStyle(element, '::before')

üè∑Ô∏è Working with Classes: className vs classList

  • element.className gets/sets the full class attribute as a string.
  • element.classList provides methods to manage individual classes:
    • add: Add a class
    • remove: Remove a class
    • toggle: Add if not present, remove if present
    • contains: Check for presence
  • classList is the modern, safer way for class manipulation.
elem.classList.add('highlight');
elem.classList.remove('active');
elem.classList.toggle('hidden');
if (elem.classList.contains('active')) { /* ... */ }

üß© Setting Multiple Styles

  • You can assign a string to element.style.cssText to set multiple inline styles at once.
  • Example:
    element.style.cssText = "color: red; background-color: yellow";

üì¶ Custom Data Attributes and Styling

  • HTML5 data attributes (e.g., data-color="blue") are accessible via element.dataset and can be used for dynamic styling logic.
  • Not directly related to styles, but often used in scripts to store style-related information.

üõ°Ô∏è Best Practices & Notes

  • Prefer using CSS classes for styling, and add/remove classes with JavaScript rather than setting styles directly.
  • Direct style changes via element.style are best for small, one-off tweaks.
  • Always check for browser compatibility for advanced CSS properties and classList in older environments.

ü¶â Senior/Lead Notes

  • Architectural Decisions: Define clear conventions for class and style management across the codebase to avoid specificity conflicts and inline style bloat.
  • Performance: Excessive DOM style updates can trigger layout thrashing. Batch changes where possible and avoid style recalculation inside animation frames or scroll events.
  • Maintainability: Relying on classes for UI state (e.g., .is-active, .is-hidden) scales better and is easier to test than manipulating inline styles.
  • Framework Integration: When working with React, Vue, etc., prefer their mechanisms for class/state management rather than direct DOM manipulation.
  • Security: If dynamic styles are based on user input, always sanitize values to prevent CSS injection attacks.
  • Tooling: Consider CSS-in-JS solutions or utility frameworks (e.g., Tailwind, Emotion) for large-scale projects to streamline dynamic styling.
