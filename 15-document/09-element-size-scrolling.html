üìê Element Size: offsetWidth/offsetHeight, clientWidth/clientHeight, scrollWidth/scrollHeight

  • offsetWidth/offsetHeight:
    - Returns the "outer" width/height of the element in pixels, including borders and padding, but NOT margins or scrollbars.
    - elem.offsetWidth / elem.offsetHeight
  • clientWidth/clientHeight:
    - Returns the "inner" width/height of the element, including padding, but NOT the border, margin, or scrollbars (except when the scrollbar is inside the element, in which case it's subtracted).
    - elem.clientWidth / elem.clientHeight
  • scrollWidth/scrollHeight:
    - Returns the entire width/height of the element content, including content not visible on the screen due to overflow.
    - elem.scrollWidth / elem.scrollHeight
Example: 
console.log(elem.offsetWidth);    // outer width with padding + border
console.log(elem.clientWidth);    // inner width with padding (no border, no scrollbar)
console.log(elem.scrollWidth);    // full content width, even the hidden part

ü™ü Getting Window Size: window.innerWidth/innerHeight

  • window.innerWidth / window.innerHeight:
    • Width/height of the entire browser window‚Äôs content area, including the scrolled out parts (viewport size).
  • document.documentElement.clientWidth/clientHeight:
    • Width/height of the HTML document‚Äôs visible part (viewport), excluding the scrollbar.
Example: 
console.log(window.innerWidth, window.innerHeight);
console.log(document.documentElement.clientWidth, document.documentElement.clientHeight);

üñ±Ô∏è Scrolling: scrollTop / scrollLeft

  • elem.scrollTop / elem.scrollLeft:
    • The number of pixels the element's content is scrolled vertically/horizontally.
    • Setting these properties scrolls the element.
    • For the <html> (or document.documentElement), these represent the scroll position of the page.
Example: 
document.documentElement.scrollTop = 100; // scrolls page 100px down
console.log(elem.scrollLeft); // get horizontal scroll position
Cross-browser Notes:
  • On some old browsers, body can also be used instead of document.documentElement, but modern browsers standardize on document.documentElement.

üìÑ Getting the Full Document Scroll Size

  • The full size of the scrollable document can be accessed by taking the maximum value of several properties:
    • scrollWidth/scrollHeight (on document.documentElement and document.body)
    • offsetWidth/offsetHeight
    • clientWidth/clientHeight
  • To get the full scrollable width/height: 
    
let scrollHeight = Math.max(
  document.body.scrollHeight, document.documentElement.scrollHeight,
  document.body.offsetHeight, document.documentElement.offsetHeight,
  document.body.clientHeight, document.documentElement.clientHeight
);
Note: The document can be scrolled to this maximum value.

‚ÜïÔ∏è Scrolling Methods: scrollTo, scrollBy, scrollIntoView

  • window.scrollTo(x, y): Scrolls the window to the given coordinates (relative to the whole document).
  • window.scrollBy(x, y): Scrolls by the given amount from the current scroll position.
  • elem.scrollIntoView(top/behavior): Scrolls the page so that the element comes into view.
    • elem.scrollIntoView(true) (or default): scrolls so that the element's top aligns with the top of the scrollable ancestor.
    • elem.scrollIntoView({behavior: "smooth"}): animates scrolling smoothly.
Example: 
window.scrollTo(0, 0); // scroll to top
window.scrollBy(0, 100); // scroll down by 100px
elem.scrollIntoView({ behavior: "smooth", block: "center" }); // smoothly center element in viewport

üßë‚Äçüíº Senior/Lead Notes

  • Performance Considerations: Excessive or continuous DOM reads/writes for size and scroll properties can trigger layout thrashing, which severely impacts performance. Batch DOM reads and writes separately.
  • Cross-browser Compatibility: Always test scroll and size behaviors on all major browsers. Some old browsers differ in their use of body vs documentElement.
  • Responsive Layouts: Be mindful that window and element sizes may change due to dynamic content, viewport resizing, or device orientation changes. Use ResizeObserver for modern reactive UI.
  • Accessibility: When using scrollIntoView, consider using options like behavior: "smooth" to improve UX, but test with screen readers and keyboard navigation.
  • Scroll Restoration: On SPA (Single Page Application) or in custom navigation, manually handle scroll position restoration for consistent UX.
  • Virtual Scrolling: For large lists/tables, consider using virtual scrolling to render only visible elements, improving performance.
  • Mobile Browsers: Scrollbar widths, appearance, and insets (like iOS safe areas) may differ. Always test on real devices.
  • Mutation Observers & Resize Observers: Use MutationObserver or ResizeObserver to respond to changes in DOM size or structure efficiently.
