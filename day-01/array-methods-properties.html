üßæ Array Properties

  • prototype: Allows extending array methods.
const arr = [1, 2, 3];
  console.log(arr.length); // 3

üìó Non-Mutating Methods (return new array or value)

  • map(): 
    [1, 2, 3].map(x => x * 2) // [2, 4, 6]
  • filter(): 
    [1, 2, 3].filter(x => x > 1) // [2, 3]
  • reduce(): 
    [1,2,3].reduce((a,b)=>a+b,0) // 6
  • reduceRight(): 
    [1,2,3].reduceRight((a,b)=>a-b) // 0
  • find(): 
    [1,2,3].find(x => x > 2) // 3
  • findIndex(): 
    [1,2,3].findIndex(x => x === 2) // 1
  • includes(): 
    [1,2,3].includes(2) // true
  • indexOf(): 
    [1,2,3].indexOf(2) // 1
  • lastIndexOf(): 
    [1,2,3,2].lastIndexOf(2) // 3
  • slice(): 
    [1,2,3].slice(1) // [2,3]
  • join(): 
    [1,2,3].join('-') // "1-2-3"
  • flat(): 
    [1,[2,3]].flat() // [1,2,3]
  • flatMap(): 
    [1,2].flatMap(x => [x, x*2]) // [1,2,2,4]
  • every(): 
    [1,2,3].every(x => x > 0) // true
  • some(): 
    [1,2,3].some(x => x > 2) // true
  • toString(): 
    [1,2,3].toString() // "1,2,3"
  • toLocaleString(): 
    [1,2,3].toLocaleString() // "1,2,3"
  • entries(): 
    [...'abc'].entries().next().value // [0, 'a']
  • keys(): 
    [...'xyz'].keys().next().value // 0
  • values(): 
    [...'ok'].values().next().value // 'o'
  • at(): 
    [10,20,30].at(-1) // 30

‚ö†Ô∏è Mutating Methods (change original array)

  • push(): 
    const a = [1]; a.push(2); // [1,2]
  • pop(): 
    a.pop() // [1]
  • shift(): 
    [1,2,3].shift() // 1
  • unshift(): 
    [2,3].unshift(1) // [1,2,3]
  • splice(): 
    [1,2,3].splice(1,1) // [1,3]
  • sort(): 
    [3,1,2].sort() // [1,2,3]
  • reverse(): 
    [1,2,3].reverse() // [3,2,1]
  • fill(): 
    [1,2,3].fill(0) // [0,0,0]
  • copyWithin(): 
    [1,2,3,4].copyWithin(1,2) // [1,3,4,4]

üìé Static & Helper Methods

  • Array.isArray(): 
    Array.isArray([1]) // true
  • Array.from(): 
    Array.from('abc') // ['a','b','c']
  • Array.of(): 
    Array.of(1,2,3) // [1,2,3]

üß† Senior/Tech Lead Notes

  • Use slice() or spread to avoid mutating original arrays.
  • Prefer immutable methods (map, filter, reduce) for predictable state.
  • Avoid splice in critical logic where immutability matters.
  • Educate team on shallow vs deep copying when working with nested structures.
