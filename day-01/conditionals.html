2. 🔀 switch / case


    
🔁 Traditional switch/case

    
function getLabel(status) {
switch (status) {
  case 'draft':
    return 'Not published';
  case 'published':
    return 'Live';
  case 'archived':
    return 'No longer available';
  default:
    return 'Unknown';
}
}


    
📦 Refactored with Lookup Object Pattern

    
const statusLabels = {
draft: 'Not published',
published: 'Live',
archived: 'No longer available'
};

function getLabel(status) {
return statusLabels[status] || 'Unknown';
}


    
🧠 Strategy Pattern using Function Map

    
const statusActions = {
    draft: () => 'Prepare content',
    published: () => 'Distribute to users',
    archived: () => 'Notify deprecation'
    };

    function executeStatusAction(status) {
    const action = statusActions[status] || (() => 'No action defined');
    return action();
    }

    

      
👨‍💼 Tech Lead Note:

      
    
        
  • ✅ Lookup Object is easier to extend and test, avoids multiple `return` or `break` statements.
    • 
        
  • ✅ Better for performance with many branches and static mappings.
    • 
        
  • ❌ Only works well for static values (not logic-heavy branches).
    • 
        
  • 🧠 Use strategy pattern for more complex logic per key.

3. 🧠 Senior / Tech Lead Insights

      
    
        
  • Guard Clauses: Use early returns to flatten control flow.
    • 
        
  • Constants / Enums: Replace magic values with constants or enums.
    • 
        
  • Named Conditions: Extract complex conditions into descriptive functions.
    • 
        
  • Minimize Duplication: DRY up repetitive logic in conditionals.
    • 
        
  • Maintainability: Choose strategies like lookup tables for extensible branching logic.
