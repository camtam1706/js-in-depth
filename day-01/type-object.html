🧠 JavaScript Objects

📦 Object Basics

Objects in JavaScript are key-value pairs. Keys are strings (or Symbols), values can be anything.

const user = { name: "Alice", age: 25 };

🔍 Dot vs Bracket Notation

  • user.name – dot notation, preferred for static keys
  • user["name"] – bracket notation, supports dynamic keys

🧮 Computed Properties

const key = "email";
const user = {
  [key]: "alice@example.com"
};

🛠️ Property Descriptors

Use Object.defineProperty() to control property behavior:

  • writable: can value be changed?
  • enumerable: shows in loops?
  • configurable: can property be redefined?

⚙️ Object Methods and this

const user = {
  name: "Alice",
  greet() {
    return `Hello, ${this.name}`;
  }
};

🔄 Cloning (Shallow vs Deep)

Shallow: Object.assign(), spread syntax. Deep: use structuredClone or custom recursion.

🧬 Prototypes and Inheritance

Objects inherit from Object.prototype. Use Object.create(proto) for custom prototype chains.

📚 Utilities & Patterns

  • Object.keys(), Object.values(), Object.entries()
  • Object.freeze(), Object.seal(), Object.defineProperty()
  • Dynamic keys, optional chaining (obj?.key), nullish coalescing (??)

🚀 Tech Lead Tips

  • Design immutable patterns with Object.freeze for shared configs.
  • Use Object.defineProperty to enforce internal rules in DDD aggregates.
  • Prefer cloning with structuredClone() in modern environments for deep copy.
  • Audit object shape in APIs, use TS interfaces or Zod schemas to prevent shape drift.
