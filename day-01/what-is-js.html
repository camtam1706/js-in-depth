What is JavaScript?

This document summarizes the core knowledge every Senior or Tech Lead JavaScript developer must deeply understand, based on the "JavaScript in Detail" course.

1. JavaScript Runtime Characteristics

  • Interpreted: JS code is parsed and executed at runtime without pre-compilation.
  • Single-threaded: All code runs on a single call stack, requiring careful async design.
  • Non-blocking: Uses event loop and callback/microtask queues to handle async I/O.

Key for Tech Leads: Must be able to explain event loop, task vs. microtask queue, and how blocking code affects performance.

2. JavaScript Engine

Runs inside browsers (e.g., Chrome's V8) or servers (Node.js). Main components:

  • Call Stack: Tracks function execution context.
  • Memory Heap: Stores variables, objects, closures.

V8 optimizes execution with JIT compilers (Ignition + TurboFan).

3. Typing System

  • Dynamic Typing: Variables can change type at runtime.
  • Weak Typing: Allows implicit type coercion.

Implication: In production, prefer TypeScript or schema validation (e.g., zod, yup) to enforce safety.

4. Execution Environment

JS runs in different environments:

  • Browser: Has access to window, DOM, localStorage, etc.
  • Node.js: Has access to fs, process, http, etc.

Lead-level expectation: Deep understanding of environment-specific globals and I/O behavior.

5. Object Model & Inheritance

  • Prototype-based: JS uses prototypes instead of class-based inheritance.
  • ES6 Classes: Are syntactic sugar over the prototype model.

Must understand: prototype chain, __proto__, Object.create, and performance cost of inheritance.

6. Standardization & Compatibility

JavaScript is defined by ECMAScript specification (ECMA-262).

  • Use transpilers (e.g., Babel) or compilers (e.g., TypeScript) for backward compatibility.
