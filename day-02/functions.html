üß© Function Declaration

Standard way to define a named function:

function greet(name) {
  return `Hello, ${name}`;
}

ü™Ñ Function Expression

Function assigned to a variable (can be anonymous):

const greet = function(name) {
  return `Hello, ${name}`;
};

üëª Anonymous Functions

Functions without a name, often used as callbacks:

setTimeout(function() {
  console.log("This runs after 1 second");
}, 1000);

Do not use parentheses when assigning a function reference:

const sayHi = function() {
  console.log("Hi");
};

const execute = sayHi; // not sayHi()
execute(); // calls sayHi

üß¨ Nested Functions

Functions can be defined inside other functions. The inner function has access to variables of the outer function (closure):

function outer(a) {
      function inner(b) {
        return a + b; // inner can access 'a'
      }
      return inner(10);
    }
    
    console.log(outer(5)); // 15

This is commonly used for encapsulation or creating closures with persistent variables.

üèπ Arrow Functions (ES6)

Arrow functions provide a concise syntax for writing functions. They do not have their own this, arguments, or super binding. 

// Basic arrow function
    const add = (a, b) => a + b;
    
    // Single parameter, no parentheses needed
    const square = x => x * x;
    
    // No parameters
    const greet = () => console.log(\"Hello\");

Arrow functions are best suited for callbacks and non-method functions.

üì¶ Function as Argument (Callback)

Functions can be passed as arguments to other functions. This enables callbacks and higher-order functions:

function process(callback) {
      console.log(\"Start\");
      callback();
      console.log(\"End\");
    }
    
    function sayHello() {
      console.log(\"Hello from callback!\");
    }
    
    process(sayHello);

This pattern is common in asynchronous code (e.g., setTimeout, event listeners, or Array methods like map()).

üîÅ Function Returning Function

A function can return another function. This pattern is useful for closures and function factories:

function multiplier(factor) {
      return function(number) {
        return number * factor;
      };
    }
    
    const double = multiplier(2);
    console.log(double(5)); // 10

The inner function \"remembers\" the factor variable from the outer function via closure.

üß† Best Practices for Senior/Tech Leads

  • Use named functions for better stack traces
  • Keep functions pure where possible (no side effects)
  • Avoid nested functions unless necessary for closures
  • Separate logic from side effects and UI handling
  • Document function parameters and return types with JSDoc or TypeScript
